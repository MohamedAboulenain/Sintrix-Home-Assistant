# Automations

> Sintrix maintains this folder. Group automation specs here when they span domains or are
> complex enough to warrant their own file. Simple per-domain automations can live in the
> domain file's **As-built** section instead.

## Spec template

```markdown
# Automation — <name>

## Intent
What it should do, and why the integrator wants it.

## Trigger / Condition / Action
Native HA constructs (see the best-practices skill). Automation mode and reason.

## As-built
ha-mcp automation id(s) created/updated. Link related domain file(s).

## Open items
```
